Guia dos Gits: O que são e utilidades

O que é o Git?

Git é um sistema de controle de versão distribuído usado para rastrear alterações em arquivos de código-fonte. Ele permite criar "snapshots" (commits) do projeto, navegar no histórico, trabalhar com branches e colaborar sem perder versões anteriores. Foi criado por Linus Torvalds em 2005 e desde então se tornou o padrão da indústria. Sua principal vantagem é a confiabilidade e flexibilidade para projetos de qualquer porte, do pessoal ao corporativo.

O que é o GitHub?

GitHub é uma plataforma online que hospeda repositórios Git. Além de armazenamento remoto (remoção/backup), oferece interface web, pull requests (revisões), issues (rastreamento), Actions (CI/CD) e ferramentas de colaboração. Ele é muito usado para projetos open source e privados, permitindo que desenvolvedores trabalhem em equipe de forma organizada e segura. O GitHub também integra recursos sociais, como perfis, estrelas e forks, tornando-se uma rede para desenvolvedores.

Diferenças entre Git e GitHub

Git: ferramenta local (controle de versão). Você pode usá-lo totalmente offline e ainda assim ter todo o histórico do projeto. É a tecnologia que controla o versionamento.

GitHub: serviço na nuvem que hospeda repositórios Git e fornece funcionalidades colaborativas. Ele facilita compartilhar seu código, colaborar com outros desenvolvedores e manter a gestão do projeto de forma centralizada.

Como ligar um projeto existente na máquina a um repositório remote

# dentro da pasta do projeto

git init

git add

git commit -m "Primeiro commit"

# adicionar repositório remoto

git remote add origin https://github.com/usuario/repositorio.git>

# enviar para o GitHub

git push -u origin main

Se a branch principal do repositório remoto for main ou master, ajuste o comando accordingly. Se o repositório remoto já existir e tiver commits, use git pull --rebase origin main antes de push para evitar conflitos. Esse processo conecta sua base de código local com o servidor do GitHub, possibilitando colaboração online.

Como gerar uma versão com Git (commit)

Veja o que mudou:

git status

Selecione os arquivos para a versão:

git add arquivo1.css arquivo2.html # ou tudo git add .

Crie o commit com mensagem:

git commit -m "mensagem clara do que mudou"

Como gerar uma versão com Git

No Git, "gerar uma versão" normalmente significa criar um commit — um snapshot do estado do código. Para versões formais também é comum usar tags. Essa prática permite identificar pontos estáveis no histórico do projeto, facilitando a distribuição de releases e correções.

add git

git commit -m "Descrição das alterações"

git tag -a v1.0.0 -m "Versão 1.0.0"

# enviar tags para o remoto

git push origin --tags

Como enviar do seu computador para o GitHub

# depois de configurar remote e comits feitos

git push origin main

# se for a primeira vez (ou quiser criar upstream)

gitgit push -u origin main

Se o repositório remoto pedir autenticação, use token de acesso pessoal (PAT) em vez de senha, ou configure SSH keys para evitar digitar credenciais sempre. Esse envio garante que seu trabalho local fique disponível na nuvem, permitindo colaboração e backup seguro.
